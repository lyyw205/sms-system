## 1. 시스템 개요

이 시스템은 게스트하우스 운영에 필요한 **3가지 핵심 업무**를 자동화합니다:

| 업무 | 수동 시 | 시스템 사용 시 |
|------|---------|---------------|
| **예약 수집** | 네이버 예약 수동 확인 → 엑셀 기록 | 5분마다 자동 동기화 |
| **객실 배정** | 예약 보고 수동으로 방 배정 | 매일 오전 10시 자동 배정 |
| **SMS 안내** | 개별 문자 수동 발송 | 템플릿 기반 시간대별 자동 발송 |

### 데이터 흐름 요약

```
네이버 예약 ──(5분 자동 동기화)──→ 예약 목록
                                    │
                          ┌─────────┴─────────┐
                          ▼                   ▼
                   객실 자동 배정         SMS 스케줄 실행
                   (매일 10시)          (설정된 시간)
                          │                   │
                          ▼                   ▼
                    방 배정 완료          문자 자동 발송
                          │                   │
                          └─────────┬─────────┘
                                    ▼
                              활동 로그 기록
```

## 3. 대시보드 — 오늘의 현황 파악

로그인 후 가장 먼저 보이는 화면입니다. **하루를 시작할 때 반드시 확인하세요.**

### 상단 통계 카드

| 카드 | 의미 | 정상 상태 |
|------|------|----------|
| **오늘의 예약** | 오늘 새로 들어온 예약 수 | 숫자 확인 |
| **오늘 발송 SMS** | 오늘 보낸 문자 수 | 0이면 스케줄 확인 필요 |
| **네이버 동기화** | 네이버 예약 자동 수집 상태 | "실행중"이 정상 |

### 7일 성별 통계 차트

- 향후 7일간 매일 남녀 게스트 비율을 보여줍니다.
- **색상 의미**:
  - 파란색: 성비 균형 (정상)
  - 노란색: 2~3:1 비율 (주의)
  - 빨간색: 3:1 초과 (경고 — 파티 규모 조정 검토)
- 파티 준비(식재료, 좌석 등)에 참고하세요.

### 하단 2열 레이아웃

| 왼쪽: 오늘의 자동발송 스케줄 | 오른쪽: 최근 예약 5건 |
|------|------|
| 각 스케줄의 이름, 발송 시간, 상태 표시 | 최근 예약자 이름, 연락처, 숙박 기간 |
| **완료**: 정상 발송됨 | 새 예약이 잘 들어오고 있는지 확인 |
| **대기**: 아직 발송 시간 전 | |
| **미발송**: 시간이 지났는데 발송 안 됨 → 확인 필요 | |

## 4. 예약 관리

### 예약이 들어오는 2가지 경로

#### 1) 네이버 자동 동기화 (주요)
- 시스템이 **5분마다** 자동으로 네이버 스마트플레이스 예약을 가져옵니다.
- 예약자명, 연락처, 숙박일, 상품명, 인원, 성별, 연령대 등이 자동 입력됩니다.
- 출처 뱃지: **네이버** (초록색)

#### 2) 수동 등록
- **예약 등록** 버튼 클릭 → 모달에서 직접 입력
- 전화 예약, 현장 방문 예약 등 네이버 외 채널용
- 출처 뱃지: **수동** (회색)

### 수동 예약 등록 방법

1. 우측 상단 **예약 등록** 버튼 클릭
2. 모달에서 입력:
   - **게스트 유형**: "미배정" (일반 객실 게스트) 또는 "파티만" (파티 전용)
   - **이름**: 예약자명 (필수)
   - **연락처**: 전화번호 (필수)
   - **체크인 날짜**: (필수)
   - **남자/여자 인원**: 각각 입력
   - **비고**: 특이사항 메모
3. **저장** 클릭

### 예약 상태

| 상태 | 뱃지 색상 | 의미 |
|------|----------|------|
| 확정 | 초록색 | 정상 예약 |
| 대기 | 노란색 | 확인 대기 중 |
| 취소 | 빨간색 | 취소된 예약 |
| 완료 | 회색 | 체크아웃 완료 |

### 예약 검색 및 필터

- **검색창**: 이름 또는 전화번호로 검색
- **날짜 필터**: 특정 날짜 예약만 보기
- **상태 필터**: 확정/대기/취소/완료 버튼 토글
- **출처 필터**: 전체/네이버/수동

### 네이버 수동 동기화

- 자동 동기화가 안 될 때: **네이버 동기화** 버튼 클릭
- 날짜를 지정하면 해당 날짜 이후 예약만 동기화
- 결과: "N건 추가, M건 업데이트" 메시지 표시

### 연박(연속 숙박) 처리

- 같은 게스트가 연속 날짜로 여러 건 예약하면 시스템이 자동으로 **연박 그룹**으로 묶습니다.
- 예약 목록에서 **연박** 뱃지가 표시됩니다.
- 연박 게스트는 같은 방에 자동 배정됩니다.
- SMS 발송 시 "연박당 1회만" 옵션으로 중복 발송을 방지합니다.

### 주의사항

- **네이버 예약은 수정/삭제 불가** — 네이버에서 변경하면 다음 동기화 시 반영
- **수동 예약만** 수정/삭제 버튼이 표시됨
- 예약 삭제 시 연결된 객실 배정과 SMS 배정도 함께 삭제됨

## 5. 객실 배정

### 자동 배정 (기본)

- **매일 오전 10시**, 시스템이 자동으로 객실을 배정합니다.
- 배정 기준:
  1. 예약 상품(네이버 biz_item)에 연결된 객실 후보 검색
  2. **여자 게스트 우선 배정** (안전 우선)
  3. 객실 우선순위(male_priority/female_priority) 순서대로 배정
  4. 연박 게스트는 이전 날짜와 같은 방 유지
  5. 도미토리: 같은 성별끼리만 배정 + 침대 수 초과 불가

### 수동 배정 (드래그 앤 드롭)

1. **객실 배정** 페이지로 이동
2. 상단에서 **날짜 선택** (좌우 화살표 또는 캘린더)
3. 탭 확인:
   - **Room**: 일반 객실 게스트
   - **미배정**: 아직 방이 없는 게스트
   - **파티**: 파티 전용 게스트
4. **미배정** 탭에서 게스트 카드를 **드래그**하여 원하는 방에 **드롭**
5. 배정 즉시 반영 + SMS 태그 자동 동기화

### 게스트 카드 정보

각 게스트 카드에는 다음 정보가 표시됩니다:
- 예약자명 (연박이면 "연박" 뱃지)
- 성별/인원 (예: 남2여1)
- 상품 유형
- 배정된 방 번호
- **SMS 칩**: 어떤 문자가 배정되었는지, 발송 여부 표시

### SMS 칩 관리

게스트 카드의 SMS 칩은 각 게스트에게 어떤 종류의 SMS가 예정/발송되었는지 보여줍니다.

| 칩 색상 | 의미 |
|---------|------|
| 회색 | 발송 대기 (아직 안 보냄) |
| 초록색 | 발송 완료 |

칩을 클릭하면:
- **발송**: 해당 SMS를 즉시 발송
- **제거**: SMS 배정 해제
- **토글**: 발송 상태 변경

### 자동 배정 수동 실행

- **자동배정** 버튼 클릭 → 미배정 게스트를 일괄 자동 배정
- 이미 수동 배정된 게스트는 건드리지 않음

### 배정 규칙 요약

| 규칙 | 설명 |
|------|------|
| 일반 객실 | 하룻밤 1예약만 가능 |
| 도미토리 | 침대 수까지 여러 예약 가능, **같은 성별만** |
| 연박 우선 | 같은 방 유지 시도 |
| 여성 우선 | 여성 게스트부터 먼저 배정 |
| 수동 보호 | 수동 배정은 자동 배정이 덮어쓰지 않음 |

## 6. 객실 설정

### 건물 관리

- 건물 추가/수정/삭제 (예: 본관, 별관)
- 정렬 순서 설정
- 활성/비활성 토글

### 객실 관리

각 객실에 설정하는 항목:

| 항목 | 설명 | 예시 |
|------|------|------|
| 방 번호 | 고유 식별자 | A101, B205 |
| 방 유형 | 객실 종류 | 더블룸, 트윈룸, 도미토리 |
| 기본/최대 수용 인원 | 인원 제한 | 2명/4명 |
| 건물 | 소속 건물 | 본관 |
| 도미토리 여부 | 다인실 플래그 | O/X |
| 침대 수 | 도미토리 침대 개수 | 8 |
| 도어 비밀번호 | 고정 비밀번호 (선택) | 1234# |
| 활성 여부 | 사용 중인 방인지 | O/X |
| 정렬 순서 | 화면 표시 순서 | 1, 2, 3... |

### 네이버 상품 연결

각 객실에 **네이버 상품(biz_item)**을 연결해야 자동 배정이 작동합니다.

1. **네이버 상품 동기화** 버튼 클릭 → 네이버에 등록된 상품 목록 가져오기
2. 객실 수정 모달에서 **네이버 상품** 다중 선택
3. 성별별 우선순위 설정 (숫자가 낮을수록 먼저 배정)

### biz_item 설정

각 네이버 상품에 기본 수용 인원과 섹션 힌트를 설정할 수 있습니다:
- 기본 수용 인원: 자동 배정 시 참고
- 섹션 힌트: room/party/unassigned 중 기본값

## 7. SMS 템플릿

### 템플릿이란?

게스트에게 보낼 문자 메시지의 **양식**입니다. 변수(`{{변수명}}`)를 사용하면 발송 시 실제 값으로 자동 치환됩니다.

### 템플릿 예시

```
{{customer_name}}님 안녕하세요!
객실 안내드립니다.

건물: {{building}}
객실: {{room_num}}
비밀번호: {{room_password}}

즐거운 여행 되세요!
```

### 사용 가능한 변수

| 변수 | 설명 | 예시 값 |
|------|------|---------|
| `{{customer_name}}` | 예약자명 | 김철수 |
| `{{room_num}}` | 배정된 방 번호 | A101 |
| `{{building}}` | 건물명 | 본관 |
| `{{room_password}}` | 객실 비밀번호 | 1234# |
| `{{naver_room_type}}` | 네이버 상품명 | 더블룸 |
| `{{participant_count}}` | 총 참가 인원 (버퍼 적용) | 25 |
| `{{male_count}}` | 남자 인원 (버퍼 적용) | 15 |
| `{{female_count}}` | 여자 인원 (버퍼 적용) | 10 |

### 참가 인원 버퍼 (중요!)

파티 안내 문자에서 인원수를 안내할 때, 실제 예약 인원에 **여유분(버퍼)**을 더해서 안내할 수 있습니다.

| 설정 | 설명 | 예시 |
|------|------|------|
| 참가자 버퍼 | 전체 인원에 더하는 수 | +5명 |
| 남자 버퍼 | 남자 인원에 더하는 수 | +2명 |
| 여자 버퍼 | 여자 인원에 더하는 수 | +3명 |
| 성비 조건부 버퍼 | 남초/여초일 때 다른 버퍼 적용 | 남초: 남+2/여+6, 여초: 남+6/여+6 |

### 반올림 설정

인원수를 깔끔한 숫자로 안내하고 싶을 때:
- **반올림 단위**: 10 (10단위로 반올림)
- **반올림 방식**: 올림(ceil) / 반올림(round) / 내림(floor)
- 예: 23명 → 올림 30명, 반올림 20명, 내림 20명

### 템플릿 관리

1. **템플릿 탭** 선택
2. **템플릿 추가** 버튼 클릭
3. 입력:
   - **템플릿 키**: 고유 식별자 (영문, 예: room_guide)
   - **이름**: 표시 이름 (예: 객실 안내)
   - **짧은 라벨**: 2~4글자 (SMS 칩에 표시, 예: 객안)
   - **내용**: 문자 본문 ({{변수}} 사용)
   - **카테고리**: 분류 (room_guide, party_guide 등)
   - **버퍼/반올림**: 필요시 설정
4. **미리보기** 버튼으로 실제 발송될 모습 확인

### 문자 유형 자동 감지

- **SMS** (단문): 한글 기준 약 45자(90바이트) 이하
- **LMS** (장문): 45자 초과 시 자동으로 장문 전환
- 비용이 다르므로 가능하면 단문으로 작성

## 8. SMS 스케줄 (자동 발송)

### 스케줄이란?

"**어떤 템플릿**을, **언제**, **누구에게** 자동으로 보낼지" 설정하는 것입니다.

### 스케줄 생성 방법

1. **스케줄 탭** 선택
2. **스케줄 추가** 버튼 클릭
3. 설정:

#### 기본 설정

| 항목 | 설명 | 예시 |
|------|------|------|
| 템플릿 | 어떤 문자를 보낼지 | 객실 안내 |
| 스케줄 이름 | 식별용 이름 | 체크인 당일 객실안내 |
| 실행 유형 | 반복 방식 | 매일/매주/간격/수동 |
| 시간 | 발송 시각 | 10시 30분 |
| 활성 | 켜기/끄기 | O |

#### 실행 유형 상세

| 유형 | 설명 | 적합한 용도 |
|------|------|------------|
| **매일(daily)** | 매일 지정 시각에 실행 | 체크인 안내 (매일 오후 2시) |
| **매주(weekly)** | 특정 요일 + 시각에 실행 | 주말 파티 안내 (금요일 오전) |
| **간격(interval)** | N분마다 반복 실행 | 실시간 안내 (30분 간격) |
| **수동(manual)** | 버튼 클릭 시에만 실행 | 비정기 공지 |

#### 대상 필터 (누구에게 보낼지)

| 필터 | 설명 | 예시 |
|------|------|------|
| **날짜 대상** | 체크인/체크아웃 기준 | 오늘 체크인, 내일 체크아웃 |
| **배정 유형** | 게스트 섹션 | room(객실), party(파티), unassigned(미배정) |
| **건물** | 특정 건물만 | 본관 게스트만 |
| **객실** | 특정 방만 | A101 게스트만 |
| **칼럼 매치** | 세부 조건 | 파티유형이 '2'인 게스트만 |
| **연박 필터** | 연박 게스트 처리 | 제외(단박만) / 포함(전부) |

#### 발송 모드

| 모드 | 설명 | 예시 |
|------|------|------|
| **1회(once)** | 체크인 날에만 1회 | 객실 안내 |
| **매일(daily)** | 숙박 기간 매일 | 매일 파티 안내 |
| **마지막 날(last_day)** | 체크아웃 전날에만 | 퇴실 안내 |
| **연박당 1회(once_per_stay)** | 연박 그룹 중 첫 날만 | 연박 할인 안내 |

#### 중복 발송 방지

- **이미 발송된 대상 제외**: 기본 ON → 같은 문자를 두 번 보내지 않음
- 스케줄이 재실행되어도 이미 받은 게스트에게는 다시 보내지 않습니다.

### 스케줄 운영

| 버튼 | 기능 |
|------|------|
| **실행** | 지금 즉시 스케줄 실행 (테스트/긴급 발송용) |
| **미리보기** | 현재 조건에 맞는 대상자 목록 확인 (발송 안 함) |
| **수정** | 설정 변경 |
| **삭제** | 스케줄 제거 |
| **활성 토글** | 스케줄 켜기/끄기 |

### 실전 예시: 일반적인 스케줄 구성

| 스케줄 | 템플릿 | 시간 | 대상 | 모드 |
|--------|--------|------|------|------|
| 체크인 객실안내 | 객실 안내 | 매일 14:00 | 오늘 체크인 + room | 1회 |
| 파티 안내 1차 | 파티 안내 | 매일 16:00 | 오늘 체크인 + party_type='2' | 매일 |
| 퇴실 안내 | 퇴실 안내 | 매일 09:00 | 오늘 체크아웃 | 마지막날 |
| 서브 객실안내 | 서브 객실 | 매일 15:00 | 오늘 체크인 + 별관 | 1회 |

## 9. 메시지 관리 (수발신)

### 화면 구성

- **왼쪽**: 연락처 목록 (최근 메시지 순)
- **오른쪽**: 선택한 연락처의 대화 내역

### 메시지 확인

1. 왼쪽 연락처 목록에서 게스트 선택
2. 오른쪽에 시간순 대화 내역 표시
3. 각 메시지에 표시되는 정보:
   - 발신/수신 방향 (말풍선 위치)
   - 시간
   - 자동응답 여부 (자동응답이면 출처 뱃지: 규칙/AI)
   - 신뢰도 % (AI 응답의 경우)
   - **검토 필요** 뱃지 (AI 신뢰도가 낮은 경우)

### 수동 문자 발송

1. 연락처 선택
2. 하단 입력창에 메시지 작성
3. **전송** 버튼 클릭
4. 글자 수/메시지 유형(SMS/LMS) 자동 표시

### "검토 필요" 메시지 처리

- AI 자동응답의 신뢰도가 60% 미만이면 **검토 필요** 뱃지가 표시됩니다.
- 해당 메시지를 확인하고:
  - 적절하면 그대로 둠
  - 부적절하면 수동으로 올바른 답변 발송

## 10. 자동응답 규칙

### 자동응답 작동 방식

게스트가 문자를 보내면 시스템이 3단계로 자동 답변을 시도합니다:

```
수신 메시지
    │
    ▼
① DB 규칙 매칭 (패턴 검색)
    │ 일치 → 자동 발송 (신뢰도 95%)
    │ 불일치 ▼
② YAML 규칙 (내장 규칙)
    │ 일치 → 자동 발송
    │ 불일치 ▼
③ AI (Claude) 판단
    │ 신뢰도 ≥ 60% → 자동 발송
    │ 신뢰도 < 60% → "검토 필요" 대기열
```

### 규칙 관리

| 항목 | 설명 | 예시 |
|------|------|------|
| 규칙 이름 | 식별용 | 위치 안내 |
| 패턴 | 정규식 (수신 메시지 매칭) | `위치\|어디\|찾아가` |
| 응답 | 자동 발송할 답변 | "○○게스트하우스 주소는..." |
| 우선순위 | 높을수록 먼저 매칭 | 10 |
| 활성 | 켜기/끄기 | O |

### 자주 사용하는 패턴 예시

| 패턴 | 매칭되는 메시지 |
|------|----------------|
| `위치\|어디\|찾아가\|주소` | "위치 알려주세요", "어디에요", "주소가?" |
| `체크인\|입실\|몇시` | "체크인 몇시에요", "입실 시간" |
| `비밀번호\|번호\|도어락` | "비밀번호 뭐에요", "도어락 번호" |
| `주차\|차\|parking` | "주차 가능한가요", "차 대수" |
| `와이파이\|wifi\|인터넷` | "와이파이 비번", "wifi 있나요" |

### 지식 베이스 문서

- **문서 탭**에서 텍스트 파일을 업로드하면 AI 자동응답의 참고 자료로 사용됩니다.
- 게스트하우스 FAQ, 시설 안내, 주변 맛집 정보 등을 업로드하세요.
- AI가 문서 내용을 참고하여 더 정확한 답변을 생성합니다.

## 11. 파티 체크인

### 용도

파티(게스트하우스 공용 이벤트)에 참가하는 게스트의 **입장 체크인**을 관리합니다.

> **스태프 계정도 사용 가능** — 모바일에 최적화된 화면

### 사용 방법

1. 상단에서 **날짜 선택** (기본: 오늘)

### 파티 유형

|------|------|------|
| 1차만 | `1` | 1차 파티만 참가 |
| 1+2차 | `2` | 1차 + 2차 모두 참가 |
| 2차만 | `2차만` | 2차 파티만 참가 |

- 파티 유형은 예약별 또는 날짜별로 다르게 설정 가능 (일별 오버라이드)

## 13. 설정 (네이버 연동)

### 네이버 쿠키 관리

네이버 스마트플레이스 예약 동기화를 위해 **네이버 로그인 쿠키**가 필요합니다.

### 연결 상태 확인

| 상태 | 의미 | 조치 |
|------|------|------|
| **연결됨** (초록) | 정상 작동 중 | 없음 |
| **연결 안됨** (빨강) | 쿠키 만료/잘못됨 | 쿠키 재설정 필요 |
| **미설정** (회색) | 쿠키가 없음 | 쿠키 입력 필요 |

### 쿠키 갱신 방법

> 쿠키는 주기적으로 만료되므로, **동기화가 안 될 때** 갱신해야 합니다.

1. PC 브라우저에서 네이버 스마트플레이스에 로그인
2. 개발자 도구(F12) → Application → Cookies에서 쿠키값 복사
3. 설정 페이지의 **쿠키 입력** 텍스트박스에 붙여넣기
4. **저장** 클릭
5. 상태가 "연결됨"으로 바뀌면 성공

### 쿠키 초기화

- **초기화** 버튼: .env 파일의 기본값으로 되돌림

## 17. 자주 묻는 질문 (FAQ)

### Q: 네이버 예약이 안 들어와요
**A**: 설정 → 네이버 연결 상태 확인. "연결 안됨"이면 쿠키를 갱신하세요. 쿠키는 주기적으로 만료됩니다.

### Q: 자동 배정이 안 된 게스트가 있어요
**A**: 다음을 확인하세요:
1. 예약 상태가 "확정"인지 확인 (대기/취소 상태는 배정 안 됨)
2. 해당 네이버 상품에 연결된 방이 있는지 (객실 설정 → 네이버 상품 연결)
3. 해당 날짜에 빈 방이 있는지 (모든 방이 차면 배정 불가)
4. 도미토리의 경우 성별이 맞는지 (남녀 혼합 불가)

### Q: 문자가 안 보내져요
**A**:
1. 대시보드에서 스케줄 상태 확인 ("미발송"이면 문제)
2. 활동 로그에서 실패 기록 확인
3. 해당 게스트에게 SMS 칩이 배정되어 있는지 확인
4. 템플릿과 스케줄이 모두 "활성" 상태인지 확인

### Q: 같은 문자가 두 번 갔어요
**A**: 스케줄의 **"이미 발송된 대상 제외"** 옵션이 꺼져 있을 수 있습니다. 스케줄 수정에서 이 옵션을 켜세요.

### Q: 연박 게스트에게 매일 같은 문자가 가요
**A**: 스케줄의 **발송 모드**를 확인하세요:
- "1회"로 설정하면 체크인 첫날만 발송
- "연박당 1회"로 설정하면 연박 그룹 중 첫 예약에만 발송

### Q: 파티 체크인 화면에 게스트가 안 보여요
**A**: 해당 게스트의 예약이 다음 조건을 충족하는지 확인:
1. 예약 상태가 "확정"
2. 해당 날짜가 숙박 기간에 포함
3. 파티 유형(party_type)이 설정되어 있음

### Q: 도미토리에 다른 성별 게스트가 배정됐어요
**A**: 수동 배정 시에는 성별 제한이 적용되지 않을 수 있습니다. 자동 배정은 반드시 같은 성별끼리만 배정합니다. 확인 후 수동으로 재배정하세요.

### Q: 비밀번호를 잊었어요
**A**: 관리자에게 문의하여 비밀번호를 재설정 받으세요. (사용자 관리 → 사용자 수정 → 새 비밀번호 입력)

## 18. 용어 사전

| 용어 | 설명 |
|------|------|
| **네이버 동기화** | 네이버 스마트플레이스 예약을 시스템으로 자동 가져오는 것 |
| **biz_item** | 네이버에 등록된 숙박 상품 (예: 더블룸, 도미토리 남성 등) |
| **템플릿** | 문자 메시지 양식 (변수 포함) |
| **스케줄** | 템플릿을 자동으로 발송하는 시간/조건 설정 |
| **SMS 칩** | 게스트 카드에 표시되는 문자 배정 상태 태그 |
| **연박** | 같은 게스트가 연속 날짜로 숙박하는 것 |
| **도미토리** | 여러 명이 함께 쓰는 다인실 (침대 단위 배정) |
| **버퍼** | 인원수에 여유분을 더하는 것 (파티 준비용) |
| **파티 유형** | 게스트의 파티 참가 범위 (1차만/1+2차/2차만) |
| **섹션** | 게스트 분류: room(객실), party(파티), unassigned(미배정) |
| **자동응답** | 수신 문자에 자동으로 답변을 보내는 기능 |
| **쿠키** | 네이버 로그인 인증 정보 (주기적 갱신 필요) |
| **LMS** | 장문 문자 (45자 초과 시 자동 전환) |
| **활동 로그** | 시스템 작업 기록 (감사 추적용) |
| **테넌트** | 멀티테넌트 시스템에서 하나의 게스트하우스 단위 |
| **반올림 모드** | 인원수를 깔끔한 숫자로 만드는 방식 (올림/반올림/내림) |
| **date_target** | 스케줄에서 오늘/내일 체크인/체크아웃 기준 선택 |
| **once_per_stay** | 연박 그룹에서 첫 예약에만 1회 발송하는 모드 |

---

> **문의**: 시스템 사용 중 문제가 발생하면 관리자에게 연락하세요.
